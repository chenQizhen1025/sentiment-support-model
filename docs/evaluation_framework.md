# 评测框架说明

## 1. 参考来源

本项目的自动评测思路参考了 `LMAPP` 项目中的评测实现，主要借鉴了以下内容：

- `BLEU-1` 到 `BLEU-4`
- `ROUGE-1`、`ROUGE-2`、`ROUGE-L`
- 自动评测与人工评测结合
- 先清洗模型输出，再做指标计算

## 2. 为什么要搭评测框架

评测框架的作用不是随便测几条回答，而是建立一套稳定的评价体系，用来比较：

- 原始模型
- `stage1` SFT 模型
- `stage2` SFT 模型
- 后续的 DPO 模型

如果没有这套评价体系，后面很难判断训练是否真的带来了提升。

## 3. 当前自动评测指标

当前自动评测保留以下指标：

- `BLEU-1`
- `BLEU-2`
- `BLEU-3`
- `BLEU-4`
- `ROUGE-1`
- `ROUGE-2`
- `ROUGE-L`
- `total_score`

其中：

- `BLEU` 更偏向看模型输出和参考答案的局部重合程度
- `ROUGE` 更偏向看整体内容覆盖度和相似度
- `total_score` 用来快速比较多个模型版本

## 4. 当前人工评测维度

为了更适合“共情支持回复模型”这个目标，人工评测建议保留以下维度：

- `共情性`
- `贴合度`
- `自然度`
- `帮助性`
- `安全性`
- `多轮连贯性`

如果是单轮评测，可以不打“多轮连贯性”。

## 5. 怎样判断训练是否优秀

对于你的项目，判断训练是否优秀，不能只看一个分数，而要综合看：

### 自动指标层面

理想表现通常是：

- `BLEU-1` 提升
- `ROUGE-1` 和 `ROUGE-L` 提升
- 综合分 `total_score` 提升

### 人工评测层面

真正重要的是这些表现有没有变好：

- 更会共情
- 更贴合来访者具体处境
- 更自然
- 更少说教
- 更安全
- 多轮里更连贯

## 6. 对你这个项目最重要的指标优先级

如果要排序，我建议优先级是：

1. `共情性`
2. `贴合度`
3. `安全性`
4. `自然度`
5. `帮助性`
6. `多轮连贯性`

也就是说，你最终不是追求“生成更长的回答”，而是追求：

- 更像支持者
- 更能接住情绪
- 更适合大学生心理支持场景
